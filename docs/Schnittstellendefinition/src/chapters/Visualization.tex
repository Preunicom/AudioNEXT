\section{Visualization-IP}

\subsection{Kurzbeschreibung}

Der Visualization-IP stellt eine AXI Lite Schnittstelle zur Verfügung.
Über diese erhält er die Lautstärke in Prozent und Grenzwerte für die farbige Visualisierung.
Das VGA Signal wird im Format 640x480@60Hz erzeugt.
Immer wenn ein Lautstärkenwert verarbeitet wurde wird ein Interrupt ausgelöst.

\subsection{Ports zum Pmod VGA}

Für die Verbindung zum Pmod VGA werden folgende Ports zur Verfügung gestellt:
\begin{enumerate}
    \item \textbf{o\_red}: roter 4-Bit-Farbkanal
    \item \textbf{o\_green}: grüner 4-Bit-Farbkanal
    \item \textbf{o\_blue}: blauer 4-Bit-Farbkanal
    \item \textbf{o\_hsync}: horizontales Synchronisationssignal \glqq HSync\grqq
    \item \textbf{o\_vsync}: vertikale Synchronisationssignal \glqq VSync\grqq
\end{enumerate}

\subsection{AXI Lite Schnittstelle}

Die Register der AXI Lite Schnittstelle werden im Abschnitt~\ref{subsec:visual_register} genauer beschrieben.

Zusammengefasst werden die bereitgestellten Lautstärkenwerte, wenn die Visualisierung aktiviert ist, auf dem Bildschirm als Text sowie als farbiger Balken dargestellt.
Nach jedem verarbeiteten Wert in VDATR wird ein Interrupt ausgelöst.
Der Wert in VDATR sollte nur überschrieben werden, wenn das FDE Flag im IPISR Register gesetzt ist, um Datenverluste zu verhindern.

Der Wert in VDATR wird in einem 7.2 Festkommaformat verlangt.
Die Bitwertigkeiten sind somit $2^7 \ 2^6 \ \dots \ 2^0 \ 2^{-1} \ 2^{-2}$.

Die Farbe des Balkens wird anhand der im COLR Register festgelegten Werte bestimmt.
Standardmäßig ist der Balken grün.
Sollte die Lautstärke die in YT gesetzte Grenze überschreiten wird der gesamte Balken gelb.
Sollte er die Grenze RT überschreiten, wird der gesamte Balken rot.
Wenn eine Grenze größer als 0x64 (100) gewählt wird, wird diese Farbe nie gesetzt.
Zusätzlich überschreibt die rote Grenze die gelbe, wodurch eine niedrigere rote als gelbe Grenze ebenfalls zur Folge hat, dass der Balken nie gelb werden kann.
